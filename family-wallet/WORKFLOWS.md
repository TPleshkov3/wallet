# Сценарии и переходы (workflow)

## 1. Регистрация и вход

| Шаг | Действие | API |
|-----|----------|-----|
| Регистрация | Создание аккаунта | `POST /auth/register` |
| Вход | Получение JWT | `POST /auth/login` |

## 2. Семья

| Шаг | Условие | API | Эффект |
|-----|---------|-----|--------|
| Создание семьи | Зарегистрирован | `POST /families` | Пользователь → `owner`, создаётся счёт `main` |
| Вступление по коду | Есть код | `POST /families/join` | Пользователь → `member` |

## 3. Счета

| Шаг | Условие | API | Эффект |
|-----|---------|-----|--------|
| Создание счёта | `owner` или `can_manage_accounts` | `POST /families/{id}/accounts` | Новый счёт |

## 4. Транзакции

| Шаг | Условие | API | Эффект |
|-----|---------|-----|--------|
| Добавление | Член семьи | `POST /transactions` | Создаётся запись, пишется лог |
| Удаление своей | Автор | `DELETE /transactions/{id}` | Удаляется, пишется лог |
| Удаление чужой | `owner` или `can_delete_any_transactions` | `DELETE /transactions/{id}` | Удаляется, пишется лог |

## 5. Регулярные платежи

| Шаг | Условие | API | Эффект |
|-----|---------|-----|--------|
| Создание | При расходе с `recurring_monthly` | `POST /transactions` | Создаётся шаблон |
| Автогенерация | При `GET /transactions` | `GET /transactions` | Проверка `day_of_month`, создание транзакций |
| Отключение | Автор или `owner` | `DELETE /transactions/recurring/{id}` | `active = false` |
| Изменение | Автор или `owner` | `PUT /transactions/recurring/{id}` | Обновление полей |

## 6. Цели накопления

| Шаг | Условие | API | Эффект |
|-----|---------|-----|--------|
| Создание/обновление | `owner` | `POST /goals` | Сохраняется цель |
| Просмотр | Член семьи | `GET /goals/current` | Возвращается прогресс |

Прогресс = `(баланс счета / целевая сумма) * 100%`

## 7. Отчёты

| Шаг | Условие | API | Эффект |
|-----|---------|-----|--------|
| Получение отчёта | Член семьи | `GET /report` | Группировка по категориям и участникам |

## 8. Права участников

| Шаг | Условие | API | Эффект |
|-----|---------|-----|--------|
| Изменение прав | `owner` | `PUT /families/{id}/permissions` | Обновление `can_manage_accounts` и `can_delete_any_transactions` |

## 9. Журнал аудита

| Шаг | Условие | API | Эффект |
|-----|---------|-----|--------|
| Просмотр логов | Член семьи | `GET /transactions/audit-log` | Последние 20 записей |

## Сводка прав

| Действие | owner | member (без прав) | member (с правами) |
|----------|-------|-------------------|-------------------|
| Создать семью | + | - | - |
| Пригласить участника | + | - | - |
| Управлять счетами | + | - | + |
| Добавить транзакцию | + | + | + |
| Удалить свою транзакцию | + | + | + |
| Удалить чужую транзакцию | + | - | + |
| Изменить права участников | + | - | - |
| Управлять целью | + | - | - |
| Смотреть отчёт | + | + | + |
