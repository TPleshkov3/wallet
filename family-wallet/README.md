# Семейный кошелек

## Краткое описание

Проект — веб-приложение для совместного учета семейных финансов. Пользователи объединяются в семью по коду приглашения, ведут доходы/расходы, работают со счетами, целями накопления, регулярными платежами и отчетами.

Ключевой акцент проекта — прозрачность и контроль прав:
- владелец управляет семьей и правами участников;
- участники видят общий бюджет, но действуют только в рамках выданных разрешений.

## Роли и авторизация

- Роли: `owner` (владелец), `member` (участник)
- Авторизация: `email/пароль`, сессия через `JWT`
- Права:
  - владелец может управлять участниками, счетами и приглашениями;
  - участникам можно выдать отдельные разрешения (`управление счетами`, `удаление чужих операций`).

## Функциональные возможности

### 1) Семьи
- создание семьи;
- приглашение по коду;
- присоединение к семье;
- список участников + роли + выданные права.

### 2) Транзакции
- добавление прихода/расхода;
- удаление своей операции;
- удаление чужих операций владельцем или участником с разрешением;
- история операций с фильтрами по дате/типу/категории.

### 3) Счета
- базовый счет `main` создается автоматически;
- можно создавать дополнительные пользовательские счета;
- все операции, отчеты и цели привязаны к выбранному счету.

### 4) Цели накопления
- одна активная цель на семью;
- параметры: название, сумма, дедлайн, счет;
- прогресс считается по формуле: `баланс счета / целевая сумма * 100%`;
- статус: «Осталось накопить» или «Цель достигнута». 

### 5) Регулярные расходы
- при создании расхода можно включить ежемесячное повторение;
- автоматическая генерация платежа в нужный день месяца;
- отдельный раздел управления шаблонами (редактирование/удаление).

### 6) Журнал изменений
- аудит ключевых действий (создание/удаление транзакций, изменение регулярных, цели и т.д.);
- последние записи доступны участникам семьи;
- события не редактируются и не удаляются пользователями.

### 7) Отчеты
- итоги за период: доходы, расходы, баланс;
- разбивка расходов:
  - по категориям (`by_category`),
  - по участникам (`by_user`: «кто сколько потратил»).

### 8) UX/UI
- SPA-интерфейс (Vue 3 + CDN);
- светлая/темная тема (сохранение в `localStorage`);
- модальные окна для добавления/удаления/редактирования;
- toast-уведомления вместо `alert`.

## API (основные эндпоинты)

### Auth
- `POST /auth/register`
- `POST /auth/login`

### Families
- `POST /families`
- `POST /families/join`
- `GET /families/me`
- `GET /families/{family_id}/invite`
- `GET /families/{family_id}/members`
- `PUT /families/{family_id}/permissions`
- `GET /families/{family_id}/accounts`
- `POST /families/{family_id}/accounts`

### Transactions & Recurring
- `POST /transactions`
- `GET /transactions`
- `DELETE /transactions/{id}`
- `GET /transactions/recurring`
- `PUT /transactions/recurring/{id}`
- `DELETE /transactions/recurring/{id}`
- `GET /transactions/audit-log`

### Goals & Reports
- `POST /goals`
- `GET /goals/current`
- `GET /report`

Swagger: `http://127.0.0.1:8000/docs`

## Стек технологий

- Backend: `FastAPI`, `SQLAlchemy`, `SQLite`, `JWT`, `bcrypt`
- Frontend: `Vue 3` (CDN), SPA без сборщика
- Тесты: `pytest`, `FastAPI TestClient`

## Запуск проекта

### 1) Backend

```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

### 2) Frontend

```bash
cd ../frontend
python3 -m http.server 5174
```

Frontend: `http://127.0.0.1:5174`  
Backend: `http://127.0.0.1:8000`

## Тестирование

```bash
cd backend
source .venv/bin/activate
pytest -q
```

## Безопасность и ограничения

- Пароли хранятся только в виде хеша (`bcrypt`)
- JWT-токен обязателен для защищенных эндпоинтов
- Проверка членства в семье на всех критичных операциях
- Ролевые проверки и granular-права на управление счетами/удаление

## Планы для развития

- миграции через Alembic;
- полноценный контроль доступа на уровне отдельных счетов;
- экспорт отчетов (CSV/PDF);
- WebSocket-обновления для операций в реальном времени.
